受講生管理システム
  　目的 　　　技術的な講座を受講する人（受講生）の個人情報や何のコースを受けたかを管理したい。  　
    要件（技術要件） 　　　　受講生の個人情報を管理できること 　　　　　　
                         受講生情報の検索、登録、更新、削除が出来ること 　　　　
                         受講生が何のコースを受けたかを管理できること 　　　　　　
                         受講生のコース情報を検索、登録、更新、削除ができること。 　　　　
                         受講生とコースの関係は1:多である。複数コース受講を考慮すること  　　　　
                         これらの情報を画面上で確認できること or 画面に連携するためのデータを返せること 　　　　
                         受講生のコース情報を一括削除できること、一括更新できること  　　　　
                         avaで開発すること 　　　　
                         SprigBootを使って開発すること 　　　　
                         （MyBatisを使って開発すること） 　　　　
                         MySQLを使うこと 　　　　
                         ドメイン駆動設計を使って開発すること 　 　
    機能 　　　　受講生情報（コース含む）を管理する機能 　　　 　
    仕様 　　　　受講生情報の定義 　　　　　
    　　　　　　　　　　　　　名前（フルネーム） 　　　　　
    　　　　　　　　　　　　　フリガナ 　　　　　
    　　　　　　　　　　　　　ニックネーム 　　　　　
    　　　　　　　　　　　　　メールアドレス 　　　　　
    　　　　　　　　　　　　　住んでる地域（市区町村まで） 　　　　　
    　　　　　　　　　　　　　電話番号（文字列として扱う） 　　　　　
    　　　　　　　　　　　　　年齢 　　　　　
    　　　　　　　　　　　　　性別（トランスジェンダー考慮） 　　　　　 　　　　
    　　　　　　　　　　　　　受講生コース情報の定義 　　　　　
    　　　　　　　　　　　　　ID 　　　　　受講生情報のID 　　　　　コース名 　　　　　受講開始日 →日付型 　　　　　
    　　　　　　　　　　　　　受講修了予定日　→日付型 　　　　 　　　　DB設計 　　　　　テーブル構成 　　　　　　
    　　　　　　　　　　　　　受講生 　　　　　　受講生コース 　　　　　　 　　　　　　
    　　　　　　　　　　　　　受講生と受講生コースは関連有り。 　　　　　　
    　　　　　　　　　　　　　1:Nの関係  受講生情報と受講生コース情報のテーブルを作成する

  受講生テーブル：students 受講生コーステーブル:students_courses
  テーブル項目の確認する構文　SHOW CREATE TABLE students_courses; TABLE以降は自分で作成したコース名を入力する。
  テーブルの中身を確認する構文　SELECT * FROM テーブル名;

  データベース名： CREATE DATABASE students_db
